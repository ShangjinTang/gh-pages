MCP Neurons and Perceptrons

This article introduces MCP neurons and perceptrons, as well as the corresponding algorithm implementation.

MCP Neuron and Perceptron Principle​

Warren McCullock and Walter Pitts studied the brain and proposed the first artificial intelligence model in 1943: the MCP neuron. Neurons are interconnected nerve cells in the brain involved in the processing and transmission of chemical and electrical signals, as shown in the figure below:

MCP neurons are simple logic gates with binary outputs. Multiple signals are transmitted from the dendrites into the cell body, and if the accumulated input signal exceeds a certain threshold, an output signal is transmitted from the axon.

Frank Rosenblatt published the perceptron learning rule based on the MCP neuron model in 1957, proposing an algorithm to automatically learn the optimal weight coefficient, which is then multiplied by the input features to decide whether to fire the neuron. This algorithm can be used for classification in supervised learning.

The perceptron takes an input vector xxx and a weight vector www, and the net input zzz is a linear combination of the two (z=w1x1+…+wmxmz = w_1 x_1 + \ldots + w_m x_mz=w1​x1​+…+wm​xm​):

w=[w1w2⋮wm],x=[x1x2⋮xm]w = \begin{bmatrix} w_1 \\ w_2 \\ \vdots \\ w_m \\ \end{bmatrix}, x = \begin{bmatrix} x_1 \\ x_2 \\ \vdots \\ x_m \\ \end{bmatrix}w=​w1​w2​⋮wm​​​,x=​x1​x2​⋮xm​​​

The activation function uses the step function ϕ(z)\phi(z)ϕ(z), θ\thetaθ is the firing threshold of the neuron:

ϕ(z)={1,if z≥θ−1,if z<θ\phi(z) = \begin{cases} 1, &\text{if } z \geq \theta \\[2ex] -1, &\text{if } z \lt \theta \end{cases}ϕ(z)=⎩⎨⎧​1,−1,​if z≥θif z<θ​

By setting x0=1x_0 = 1x0​=1 and w0=−θw_0 = -\thetaw0​=−θ, integrating θ\thetaθ into zzz can make the above equation neater:

z=w0x0+w1x1+w2x2+…+wmxm=wTxz = w_0 x_0 + w_1 x_1 + w_2 x_2 + \ldots + w_m x_m = w^T xz=w0​x0​+w1​x1​+w2​x2​+…+wm​xm​=wTx ϕ(z)={1,if z≥0−1,if z<0\phi(z) = \begin{cases} 1, &\text{if } z \geq 0 \\[2ex] -1, &\text{if } z \lt 0 \end{cases}ϕ(z)=⎩⎨⎧​1,−1,​if z≥0if z<0​

The following figure shows the activation function that obtains 1 or -1 through z=wTxz = w^T xz=wTx (left subfigure), and its use to distinguish two linearly separable classes (right subfigure):

The idea of the MCP neuron and Rosenblatt threshold perceptron models is to simplify the simulation of the working principle of a single neuron in the brain, that is, whether the input reaches the triggering condition.

The Rosenblatt perceptron rule is very simple:

  1. Initialize the weights to 0 or a small random number.
  2. For each training sample x(i)x^{(i)}x(i), perform the following steps:
    1. Calculate the output value y^\hat yy^​.
    2. Update weights.

The update of weight wjw_jwj​ can be expressed as:

wj:=wj+Δwjw_j := w_j + \Delta w_jwj​:=wj​+Δwj​

The jjjth weight Δwj\Delta w_jΔwj​ used to update the weight vector can be calculated by the perceptron learning rule:

Δwj=η(y(i)−y^(i))xj(i)\Delta w_j = \eta (y^{(i)} - \hat y^{(i)}) x_j^{(i)}Δwj​=η(y(i)−y^​(i))xj(i)​

where η\etaη is the learning rate (between 0 and 1), y(i)y^{(i)}y(i) is the actual class label for the iiith training, y^(i)\hat y^{(i)}y^​(i) is the predicted class label for the iiith training. It should be noted that all weights in the weight vector are updated at the same time, and the next y^(i)\hat y^{(i)}y^​(i) will be updated only after the weights are updated.

For example, the weights of two-dimensional data are updated as follows:

Δw0=η(y(i)−output(i))Δw1=η(y(i)−output(i))x1(i)Δw2=η(y(i)−output(i))x2(i)\begin{align} \Delta w_0 &= \eta (y^{(i)} - \text{output}^{(i)}) \\ \Delta w_1 &= \eta (y^{(i)} - \text{output}^{(i)}) x_1^{(i)} \\ \Delta w_2 &= \eta (y^{(i)} - \text{output}^{(i)}) x_2^{(i)} \end{align}Δw0​Δw1​Δw2​​=η(y(i)−output(i))=η(y(i)−output(i))x1(i)​=η(y(i)−output(i))x2(i)​​​

In the case where the perceptron predicts the correct class label, the weights remain unchanged:

Δwj=η(−1(i)−−1(i))xj(i)=0Δwj=η(1(i)−1(i))xj(i)=0\begin{align} \Delta w_j &= \eta (-1^{(i)} - -1^{(i)}) x_j^{(i)} = 0 \\ \Delta w_j &= \eta (1^{(i)} - 1^{(i)}) x_j^{(i)} = 0 \end{align}Δwj​Δwj​​=η(−1(i)−−1(i))xj(i)​=0=η(1(i)−1(i))xj(i)​=0​​

In the case where the perceptron predicts the wrong class label, the weights are updated in the direction of the correct class label:

2Δwj=η(1(i)−−1(i))xj(i)=η(2)xj(i)Δwj=η(−1(i)−1(i))xj(i)=η(−2)xj(i)\begin{align}{2} \Delta w_j &= \eta (1^{(i)} - -1^{(i)}) x_j^{(i)} = \eta (2) x_j^{(i)} \\ \Delta w_j &= \eta (-1^{(i)} - 1^{(i)}) x_j^{(i)} = \eta (-2) x_j^{(i)} \end{align}2Δwj​Δwj​​=η(1(i)−−1(i))xj(i)​=η(2)xj(i)​=η(−1(i)−1(i))xj(i)​=η(−2)xj(i)​​​

In order to better understand the role of the multiplier factor xj(i)x_j^{(i)}xj(i)​ in the above expression, here is an example based on the following conditions:

y(i)=1,y^(i)=−1,η=1y^{(i)} = 1, \hat y^{(i)} = -1, \eta = 1y(i)=1,y^​(i)=−1,η=1

If there is a sample xj(i)=0.5x_j^{(i)} = 0.5xj(i)​=0.5, but it is misclassified as -1, the corresponding Δwj\Delta w_jΔwj​ is:

Δwj(i)=1×(1−(−1))×0.5=1\Delta w_j^{(i)} = 1 \times (1 - (-1)) \times 0.5 = 1Δwj(i)​=1×(1−(−1))×0.5=1

If there is another sample with xj(i)=2x_j^{(i)} = 2xj(i)​=2, but it is misclassified as -1, a larger Δwj\Delta w_jΔwj​ will be generated so that it can be classified correctly next time:

Δwj(i)=1×(1−(−1))×2=4\Delta w_j^{(i)} = 1 \times (1 - (-1)) \times 2 = 4Δwj(i)​=1×(1−(−1))×2=4

Note that the perceptron can only converge when the classes are linearly separable and the learning rate is small enough. If the classes are not linearly separable, you need to set the maximum number of training times or set the tolerable number of misclassifications, otherwise the perceptron will always update the weights:

The following figure briefly describes the principle of the perceptron:

In the input stage, the perceptron receives the input of sample xxx and combines it with weight www to calculate the net input, and then passes the net input to the activation function (here using a unit step function), which generates the output predicted class label -1 or +1. During the learning phase, this output is used to calculate the prediction error and update the weights.

Python Implementation of Rosenblatt Perceptron​

In the following code, an object-oriented approach is used to define the perceptron as a Python class.

import numpy as np


classPerceptron(object):
     def __init__(self, eta=0.01, n_iter=50, random_state=1):
         self.eta = eta
         self.n_iter = n_iter
         self.random_state = random_state

     def fit(self, X, y):
         rgen = np.random.RandomState(self.random_state)
         self.w_ = rgen.normal(loc=0.0, scale=0.01, size=1 + X.shape[1])
         self.errors_ = []
         for _ in range(self.n_iter):
             errors = 0
             for xi, target in zip(X, y):
                 errors += int(self.predict(xi) != target)
                 update = self.eta * (target - self.predict(xi))
                 self.w_[1:] += update * xi
                 self.w_[0] += update
             self.errors_.append(errors)
         return self

     def net_input(self, X):
         return np.dot(X, self.w_[1:]) + self.w_[0]

     def predict(self, X):
         return np.where(self.net_input(X) >= 0.0, 1, -1)

With the above implementation, the perceptron object can be initialized by specifying the learning rate eta and the number of training times n_iter. The fit method can initialize an all-0 weight vector self.w_, containing the weights w0w_0w0​ ~ wmw_mwm​.

After the weights are initialized, the weights are updated using the perceptron's learning rules. fit passes in known samples and corresponding labels, which are used in the training phase to obtain algorithm parameters; predict is used to predict the corresponding labels of any samples (usually unknown samples are used to observe algorithm performance), and is used in the verification phase.

In addition, the number of incorrectly classified samples self.errors_ is defined here to observe the performance of the algorithm after each training.

Train Perceptron Model​

Next we train our perceptron algorithm on the extracted subset of Iris data. At the same time, we will plot the classification error after each training to check whether the algorithm converges and find the decision boundary that separates the two flower classes:

ppn = Perceptron(eta=0.1, n_iter=10)
ppn.fit(X, y)

plt.plot(range(1, len(ppn.errors_) + 1), ppn.errors_, marker="o")

It can be observed from the above figure that after six trainings, the number of sample misclassifications is 0 and the algorithm has converged.

Decision Boundary Drawing​

Define the decision boundary drawing function as follows:

from matplotlib.colors import ListedColormap


def plot_decision_regions(X, y, classifier, resolution=0.02):
    # setup marker generator and color map
    markers = ("s", "x", "o", "^", "v")
    colors = ("red", "blue", "lightgreen", "gray", "cyan")
    cmap = ListedColormap(colors[: len(np.unique(y))])

    # plot the decision surface
    x1_min, x1_max = X[:, 0].min() - 1, X[:, 0].max() + 1
    x2_min, x2_max = X[:, 1].min() - 1, X[:, 1].max() + 1
    xx1, xx2 = np.meshgrid(
        np.arange(x1_min, x1_max, resolution), np.arange(x2_min, x2_max, resolution)
    )
    Z = classifier.predict(np.array([xx1.ravel(), xx2.ravel()]).T)
    Z = Z.reshape(xx1.shape)
    plt.contourf(xx1, xx2, Z, alpha=0.4, cmap=cmap)
    plt.xlim(xx1.min(), xx1.max())
    plt.ylim(xx2.min(), xx2.max())

    # plot class samples
    for idx, cl in enumerate(np.unique(y)):
        plt.scatter(
            x=X[y == cl, 0],
            y=X[y == cl, 1],
            alpha=0.8,
            c=cmap(idx),
            edgecolor="black",
            marker=markers[idx],
            label=cl,
        )

Plot the sample from the previous section:

plot_decision_regions(X, y, classifier=ppn)
plt.xlabel("sepal length [cm]")
plt.ylabel("petal length [cm]")
plt.legend(loc="upper left")

plt.tight_layout()
plt.show()

Through learning, the perceptron generates decision boundaries that completely classify the Iris training subset.

note

Although the perceptron in this example is able to completely classify both flowers, convergence is still one of the most important issues for the perceptron. If the two classes are linearly separable, the perceptron algorithm can converge. However, for classes that are not linearly separable, the weights will never stop updating unless we manually set the maximum number of training times.
