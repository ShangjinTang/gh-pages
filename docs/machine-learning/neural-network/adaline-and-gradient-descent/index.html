Adaline and Gradient Descent

This article introduces the Adaptive Linear Neuron (Adaline) based on the gradient descent method (Gradient Descent, GD) and the corresponding algorithm implementation.

Adaline Principle​

Adaline was proposed by Bernard Widrow and Tedd Hoff in 1960 and was considered an improvement on the Rosenblatt perceptron. The Adaline algorithm defines the core concept of minimizing the cost function and lays the foundation for some of the more advanced machine learning algorithms (such as logistic regression and support vector machines).

The difference between the Adaline rule and the Rosenblatt perceptron lies in the activation function used to adjust the weights. The former uses a linear function and the latter uses a unit step function. In Adaline, ϕ(wTx)=wTx\phi(w^T x)= w^T xϕ(wTx)=wTx.

In addition to using a linear activation function to update the weights, Adaline uses a unit step quantizer to predict the class label, as shown in the following figure:

Batch Gradient Descent​

Defining the objective function that needs to be optimized during the learning process is a key step in supervised learning. The objective function is usually a cost function that needs to be minimized. For Adaline, we define the cost function JJJ of the weights as the sum of variances (Sum of Squared Errors, SSE) between the predicted class label and the actual class label.

J(w)=12Σi(y(i)−ϕ(z(i)))2\begin{align} J(w) = {\frac 1 2} \Sigma_i(y^{(i)} - \phi(z^{(i)}))^2 \end{align}J(w)=21​Σi​(y(i)−ϕ(z(i)))2​​

The advantage of a linear activation function is that the cost function is differentiable and convex (the local minimum is the global minimum). ​

∂J(w)∂wj=12Σi(y(i)−ϕ(z(i)))2∂z∂z∂wj=−Σi(y(i)−ϕ(z(i)))xj(i)\begin{align} {\frac {\partial J(w)} {\partial w_j}} &= {\frac 1 2} {\frac {\Sigma_i(y^{(i)} - \phi(z^{(i)}))^2} {\partial z}} {\frac {\partial z} {\partial w_j}} \\ &= -{\Sigma_i(y^{(i)} - \phi(z^{(i)}))} x_j^{(i)} \end{align}∂wj​∂J(w)​​=21​∂zΣi​(y(i)−ϕ(z(i)))2​∂wj​∂z​=−Σi​(y(i)−ϕ(z(i)))xj(i)​​​

An algorithm called gradient descent is used to find the weights that minimize the cost function in order to classify samples from a subset of the Iris data. As shown in the figure below, gradient descent can be analogized to the process of going down a mountain until a local minimum is reached. Each training session is equivalent to walking one step on the slope, where the step size is determined by the learning rate and slope:

The idea of the gradient descent method is that if the function is differentiable and defined at a certain point, then the function will decrease fastest along the opposite direction of the gradient at that point, and the decrease coefficient is represented by the learning rate η\etaη:

Δwj=−ηΔJ(w)\begin{align} \Delta w_j = -\eta \Delta J(w) \end{align}Δwj​=−ηΔJ(w)​​

After each training, the weights are updated as:

wj:=wj+Δwj=wj−ηΔJ(w)\begin{align} w_j := w_j + \Delta w_j = w_j - \eta \Delta J(w) \end{align}wj​:=wj​+Δwj​=wj​−ηΔJ(w)​​

Each update of the weight is based on all samples in the training set (rather than a single sample), so this most primitive gradient descent method is also called Batch Gradient Descent (BGD).

Adaline Implementation based on Batch Gradient Descent​

In the code below, an object-oriented approach is used to define Adaline as a Python class.

import numpy as np


class AdalineGD(object):
    def __init__(self, eta=0.01, n_iter=50, random_state=1):
        self.eta = eta
        self.n_iter = n_iter
        self.random_state = random_state

    def fit(self, X, y):
        rgen = np.random.RandomState(self.random_state)
        self.w_ = rgen.normal(loc=0.0, scale=0.01, size=1 + X.shape[1])
        self.cost_ = []

        for i in range(self.n_iter):
            net_input = self.net_input(X)
            output = self.activation(net_input)
            errors = y - output
            self.w_[1:] += self.eta * X.T.dot(errors)
            self.w_[0] += self.eta * errors.sum()
            cost = (errors ** 2).sum() / 2.0
            self.cost_.append(cost)
        return self

    def net_input(self, X):
        return np.dot(X, self.w_[1:]) + self.w_[0]

    def activation(self, X):
        return X

    def predict(self, X):
        return np.where(self.activation(X) >= 0.0, 1, -1)

Stochastic Gradient Descent​

The batch gradient descent method needs to use all samples every time it is trained. Assuming there is a data set on the order of millions (which is not uncommon in machine learning), using the batch gradient descent method will bring huge computational costs, and the weights are updated as follows:

Δw=ηΣi(y(i)−ϕ(z(i)))x(i)\begin{align} \Delta w = \eta {\Sigma_i(y^{(i)} - \phi(z^{(i)}))} x^{(i)} \end{align}Δw=ηΣi​(y(i)−ϕ(z(i)))x(i)​​

A popular alternative is Stochastic Gradient Descent (SGD), sometimes called iterative or on-line gradient descent, which updates the weights using a single training example at a time. :

Δw=η(y(i)−ϕ(z(i)))x(i)\begin{align} \Delta w = \eta (y^{(i)} - \phi(z^{(i)})) x^{(i)} \end{align}Δw=η(y(i)−ϕ(z(i)))x(i)​​

Compared with the batch gradient descent method, the stochastic gradient descent method updates the weights more frequently and usually converges faster. Since a single sample is highly random (that is, noisy), each iteration direction is not always in the direction of the local optimal solution, and the accuracy of the algorithm will decrease relatively. However, as the number of training increases, it can usually converge to the vicinity of the local optimal solution. When using stochastic gradient descent, the samples need to be shuffled at each training epoch to obtain random samples.

In the implementation of stochastic gradient descent, the fixed learning rate η\etaη is often replaced by an adaptive learning rate that decreases over time, such as num1niter+num2{\frac {num1} {n_{iter} + num2} }niter​+num2num1​, where num1num1num1 and num2num2num2 are constants.

Another advantage of the stochastic gradient descent method is online learning (On-line Learning). Whenever new sample data is obtained, the model can be trained. This can be applied when large amounts of data are continuously input (such as user data in web applications), and samples can be discarded after each training to prevent insufficient space caused by data accumulation.

Mini-batch Gradient Descent (MBGD) is a fusion of the ideas of batch gradient descent and stochastic gradient descent. Each training uses a small batch of samples (for example, the total number of samples is 1000, and each training uses random ones. 10), which can overcome the problems of slow BGD training and large noise of SGD.

Adaline Implementation based on Stochastic Gradient Descent​

Adjust the code in the previous section and add the weight online learning function partial_fit, the training sample shuffling function _shuffle, the weight initialization function _initialize_weights and the weight adjustment function _update_weights to implement Adaline based on stochastic gradient descent.

import numpy as np


class AdalineSGD(object):
    def __init__(self, eta=0.01, n_iter=10, shuffle=True, random_state=None):
        self.eta = eta
        self.n_iter = n_iter
        self.w_initialized = False
        self.shuffle = shuffle
        self.random_state = random_state

    def fit(self, X, y):
        self._initialize_weights(X.shape[1])
        self.cost_ = []
        for i in range(self.n_iter):
            if self.shuffle:
                X, y = self._shuffle(X, y)
            cost = []
            for xi, target in zip(X, y):
                cost.append(self._update_weights(xi, target))
            avg_cost = sum(cost) / len(y)
            self.cost_.append(avg_cost)
        return self

    def partial_fit(self, X, y):
        if not self.w_initialized:
            self._initialize_weights(X.shape[1])
        if y.ravel().shape[0] > 1:
            for xi, target in zip(X, y):
                self._update_weights(xi, target)
        else:
            self._update_weights(X, y)
        return self

    def _shuffle(self, X, y):
        r = self.rgen.permutation(len(y))
        return X[r], y[r]

    def _initialize_weights(self, m):
        """Initialize weights to small random numbers"""
        self.rgen = np.random.RandomState(self.random_state)
        self.w_ = self.rgen.normal(loc=0.0, scale=0.01, size=1 + m)
        self.w_initialized = True

    def _update_weights(self, xi, target):
        output = self.activation(self.net_input(xi))
        error = target - output
        self.w_[1:] += self.eta * xi.dot(error)
        self.w_[0] += self.eta * error
        cost = 0.5 * error ** 2
        return cost

    def net_input(self, X):
        return np.dot(X, self.w_[1:]) + self.w_[0]

    def activation(self, X):
        return X

    def predict(self, X):
        return np.where(self.activation(self.net_input(X)) >= 0.0, 1, -1)
