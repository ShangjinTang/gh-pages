Preprocessing of Categorical Data

In practice, collected data are often incomplete, inconsistent, and may contain many errors. Data Preprocessing is a data mining technology that processes raw data for further analysis.

This article introduces the processing of Categorical Data.

Classification and ordering​

When processing categorical data, it is necessary to distinguish between nominal (nominal) features and ordinal (ordinal) features.

  • Classification characteristics: Different categories, comparison with each other is meaningless. Such as name, gender, fruits, etc.
  • Sequencing features: Different categories can be compared and sorted with each other. Such as very satisfied/generally satisfied/dissatisfied, small/medium/large, etc. Unlike numerical features, the difference between the two is generally meaningless.

The following df variables represent some characteristics of the T-shirt:

>>> import pandas as pd
>>> df = pd.DataFrame([
... ['green', 'M', 10.1, 'class1'],
... ['red', 'L', 13.5, 'class2'],
... ['blue', 'XL', 15.3, 'class1']])
>>> df.columns = ['color', 'size', 'price', 'classlabel']
>>> df
    color size price classlabel
0 green M 10.1 class1
1 red L 13.5 class2
2 blue XL 15.3 class1

These include the categorical feature color(color), the ordinal feature size(size) and the numeric feature price(price). The last column is the classification category label.

Mapping of ordinal features​

In order to ensure that the learning algorithm can recognize the ordered features, it is necessary to manually map the classification string to an integer.

As for the T-shirt size in the above example, assuming that the order XL>L>MXL > L > MXL>L>M is known, the following conversion can be performed:

>>> size_mapping = {
...'XL': 3,
...'L': 2,
...'M': 1}
>>> df['size'] = df['size'].map(size_mapping)
>>> df
    color size price classlabel
0 green 1 10.1 class1
1 red 2 13.5 class2
2 blue 3 15.3 class1

For reverse transformation, create a reverse dictionary and map:

>>> inv_size_mapping = {v: k for k, v in size_mapping.items()}
>>> df['size'].map(inv_size_mapping)
0 M
1L
2XL
Name: size, dtype: object

Encoding of class labels​

Many machine learning libraries require the class label Encoding to be an integer value; although scikit-learn has integrated this processing mechanism by default, it is recommended to get into the habit of manual conversion.

The numeric size of a class label has no meaning, so you can directly use an enumeration for label conversion:

>>> import numpy as np
>>> class_mapping = {label:idx for idx,label in
... enumerate(np.unique(df['classlabel']))}
>>> class_mapping
{'class1': 0, 'class2': 1}

Encode class labels as integers:

>>> df['classlabel'] = df['classlabel'].map(class_mapping)
>>> df
    color size price classlabel
0 green 1 10.1 0
1 red 2 13.5 1
2 blue 3 15.3 0

Reverse conversion:

>>> inv_class_mapping = {v: k for k, v in class_mapping.items()}
>>> df['classlabel'] = df['classlabel'].map(inv_class_mapping)
>>> df
    color size price classlabel
0 green 1 10.1 class1
1 red 2 13.5 class2
2 blue 3 15.3 class1

Class labels can be encoded as integers more easily via sklearn.preprocessing.LabelEncoder:

>>> from sklearn.preprocessing import LabelEncoder
>>> class_le = LabelEncoder()
>>> y = class_le.fit_transform(df['classlabel'].values)
>>> y
array([0, 1, 0])

Reverse conversion:

>>> class_le.inverse_transform(y)
array(['class1', 'class2', 'class1'], dtype=object)

One-hot encoding of nominal features​

The principle of one-hot encoding​

Before introducing One-Hot Encoding (One-Hot Encoding), let me first explain why the encoding method in the previous chapter is not used.

If you encode it as before:

>>> X = df[['color', 'size', 'price']].values
>>> color_le = LabelEncoder()
>>> X[:, 0] = color_le.fit_transform(X[:, 0])
>>>X
array([[1, 1, 10.1],
        [2, 2, 13.5],
        [0, 3, 15.3]], dtype=object)

The encoding result is:

  • blue = 0
  • green = 1
  • red = 2

If you feed the above data to a classifier, one of the most common mistakes in dealing with categorical data occurs: although we know that the numbers 0, 1, and 2 do not represent magnitudes, the algorithm does not know that. Therefore, during the learning process, the algorithm will associate them by default, that is, assuming red > green > blue. After this processing, the algorithm can still produce certain results, but its performance will be affected.

The idea of one-hot encoding is to create a new feature for each value. For the above example, the color features can be converted into three new features: blue, green and red, and then labeled with binary values. For the blue sample, the encoding is blue=1 , green=0 , red=0 .

Implementation of one-hot encoding​

Use sklearn.preprocessing.OneHotEncoder to encode the feature color and return a sparse matrix:

>>> from sklearn.preprocessing import OneHotEncoder
>>> ohe = OneHotEncoder(categorical_features=[0])
>>> ohe.fit_transform(X).toarray()
array([[ 0. , 1. , 0. , 1. , 10.1],
        [0., 0., 1., 2., 13.5],
        [ 1. , 0. , 0. , 3. , 15.3]])

Another more convenient one-hot encoding method is the get_dummies method in pandas, which converts the specified string column of the DataFrame, leaving other columns unchanged:

>>> pd.get_dummies(df[['price', 'color', 'size']])
    price size color_blue color_green color_red
0 10.1 1 0 1 0
1 13.5 2 0 0 1
2 15.3 3 1 0 0

Correlation of one-hot encoding​

When using popular coding data sets, you must remember that it introduces multicollinearity, that is, one variable can be linearly predicted by other variables (such as the matrix above, if blue, green, red are known Any two of them can get the last one). This has an impact on certain operations such as matrix inversion.

To reduce the correlation between variables, we can simply remove a feature column from the one-hot encoding array.

sklearn.preprocessing.OneHotEncoder does not provide a feature column deletion method and needs to be converted into a numpy array before slicing:

>>> ohe = OneHotEncoder(categorical_features=[0])
>>> ohe.fit_transform(X).toarray()[:, 1:]
array([[ 1. , 0. , 1. , 10.1],
[0., 1., 2., 13.5],
[ 0. , 0. , 3. , 15.3]])

get_dummies in pandas provides the parameter drop_first, which can easily delete the first feature column:

>>> pd.get_dummies(df[['price', 'color', 'size']],
... drop_first=True)
   price  size  color_green  color_red
0   10.1     1            1          0
1   13.5     2            0          1
2   15.3     3            0          0
