Confusion Matrix

In addition to accuracy, evaluation metrics also include other indicators, such as Precision, Recall, and F1 Score. They are evaluation metrics in confusion matrix.

Concept of Confusion Matrix​

Before introducing the evaluation indicators of different scoring indicators, let us first introduce the confusion matrix (Confusion Matrix):

  • TP (True Positives): The number of samples predicted to be positive and actually positive
  • FN (False Negatives): Number of samples predicted to be negative but actually positive
  • FP (False Positives): the number of samples predicted to be positive but actually negative
  • TN (True Negatives): The number of samples that are predicted to be negative but actually are negative

The confusion matrix is of the form True/False + Positive/Negative (Positive/Negative is for the predicted class label).

Confusion Matrix Using Scikit-learn​

Take the wdbc data set as an example to illustrate. Import library and do data preprocessing:

import pandas as pd
from sklearn.preprocessing import LabelEncoder
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.svm import SVC
from sklearn.pipeline import make_pipeline

df = pd.read_csv("https://archive.ics.uci.edu/ml/machine-learning-databases/breast-cancer-wisconsin/wdbc.data",
                  header=None)
X = df.iloc[:, 2:]
y = df.iloc[:, 1]
le = LabelEncoder()
y = le.fit_transform(y)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2, stratify=y, random_state=1)

pipe_svc = make_pipeline(StandardScaler(), SVC(random_state=1))

Calculate the confusion matrix via sklearn.metrics.confusion_matrix:

from sklearn.metrics import confusion_matrix

pipe_svc.fit(X_train, y_train)
y_pred = pipe_svc.predict(X_test)
confmat = confusion_matrix(y_true=y_test, y_pred=y_pred)
print(confmat)

The above function prints out the confusion matrix of pipe_svc on the test set:

It should be noted that the confusion matrix arrangement of scikit-learn is [[TN, FP], [FN, TP]].

[[71 1]
  [2 40]]

To illustrate more intuitively, plot with matplotlib as follows:

import matplotlib.pyplot as plt

fig, ax = plt.subplots(figsize=(2.5, 2.5))
ax.matshow(confmat, cmap=plt.cm.Blues, alpha=0.3)
for i in range(confmat.shape[0]):
     for j in range(confmat.shape[1]):
         ax.text(x=j, y=i, s=confmat[i, j], va='center', ha='center')
plt.xlabel('predicted label')
plt.ylabel('true label')
plt.show()

The class label of this data set is malignant or benign. The classification label is encoded through LabelEncoder. Malignant is 1 (positive) and benign is 0 (negative).

  • Top left: TN, the number of samples predicted to be benign and correctly predicted.
  • Top right: FP, the number of samples predicted to be malignant and incorrectly predicted (actually benign).
  • Bottom left: FN, the number of samples predicted to be benign and incorrectly predicted (actually malignant).
  • Bottom right: TP, the number of samples predicted to be malignant and correctly predicted.

Although in most cases the focus is on the performance of correct classification (TN + TP). But for some practical problems, FP and FN are different. Based on the above wdbc data, if a benign condition is predicted to be malignant (FP), the patient is likely to undergo a follow-up reexamination; but if a malignant condition is predicted to be benign (FN), the patient may take it lightly and delay treatment. It will have more serious consequences than FP. This is also the reason why different evaluation indicators need to be developed.

For the convenience of memory, FP is also called false alarm, which can be understood as a false alarm of the alarm bell; FN is also called miss, which should alarm but does not actually alarm.

Introduction to evaluation metrics​

The following picture comes from Wikipedia - Confusion matrix:

Some important evaluation indicators are introduced below, where P=TP+FNP = TP + FNP=TP+FN represents the number of samples that are actually positive, and N=TN+FPN = TN + FPN=TN+FP represents the number of samples that are actually negative.

Accuracy​

ACC (Accuracy) represents accuracy and ERR (Error) represents error rate:

ACC=TP+TNP+N=TP+TNTP+TN+FP+FNERR=FP+FNP+N=FP+FNTP+TN+FP+FN=1−ACC\begin{align} ACC & = \frac {TP + TN} {P + N} = \frac {TP + TN} {TP + TN + FP + FN} \\ ERR & = \frac {FP + FN} {P + N} = \frac {FP + FN} {TP + TN + FP + FN} = 1 - ACC \end{align}ACCERR​=P+NTP+TN​=TP+TN+FP+FNTP+TN​=P+NFP+FN​=TP+TN+FP+FNFP+FN​=1−ACC​​

Implement in scikit-learn as follows:

from sklearn.metrics import accuracy_score

accuracy = accuracy_score(y_true=y_test, y_pred=y_pred)

Recall​

Recall rate is TPR (True Positive Rate), which represents the probability of correct prediction in actual positive samples, also known as Sensitivity or Hit Rate:

Recall=TPR=TPP=TPTP+FN\begin{align} Recall = TPR = \frac {TP} {P} = \frac {TP} {TP + FN} \end{align}Recall=TPR=PTP​=TP+FNTP​​​

Implement in scikit-learn as follows:

from sklearn.metrics import recall_score

recall = recall_score(y_true=y_test, y_pred=y_pred)

Precision​

Precision rate is PPV（Positive Predictive Value）, which represents the probability of correct prediction in samples with positive predictions:

Precision=PPV=TPTP+FP\begin{align} Precision = PPV = \frac {TP} {TP + FP} \end{align}Precision=PPV=TP+FPTP​​​

Implement in scikit-learn as follows:

from sklearn.metrics import precision_score

precision = precision_score(y_true=y_test, y_pred=y_pred)

F1 score​

The F1 Score is the harmonic mean of recall and precision:

F1=21Recall+1Precision=2Recall×PrecisionRecall+Precision\begin{align} F_1 = \cfrac 2 {\cfrac 1 {Recall} + \cfrac 1 {Precision}} = 2 \frac {Recall \times Precision} {Recall + Precision} \end{align}F1​=Recall1​+Precision1​2​=2Recall+PrecisionRecall×Precision​​​

Implement in scikit-learn as follows:

from sklearn.metrics import f1_score

f1 = f1_score(y_true=y_test, y_pred=y_pred)

Matthews correlation coefficient​

Matthews Correlation Coefficient (MCC) comprehensively scores multiple indicators:

MCC=TP×TN−FP×FN(TP+FP)(TP+FN)(TN+FP)(TN+FN)\begin{align} MCC = \frac {TP \times TN - FP \times FN} {\sqrt{(TP + FP)(TP + FN)(TN + FP)(TN + FN)}} \end{align}MCC=(TP+FP)(TP+FN)(TN+FP)(TN+FN)​TP×TN−FP×FN​​​

Implement in scikit-learn as follows:

from sklearn.metrics import matthews_corrcoef

mcc = matthews_corrcoef(y_true=y_test, y_pred=y_pred)

Replace Evaluation Metrics in Scikit-learn​

It should be noted that most algorithms in scikit-learn use accuracy_score as the evaluation index by default. In addition, the positive class is considered to be the class with label 1.

You can replace the evaluation metrics and the class labels of the positive classes through sklearn.metrics.make_scorer:

from sklearn.metrics import make_scorer, f1_score

scorer = make_scorer(f1_score, pos_label=0)
gs = GridSearchCV(estimator=pipe_svc, param_grid=param_grid, scoring=scorer, cv=10)
gs = gs.fit(X_train, y_train)
print(gs.best_score_)
# Output: 0.986202145696
>>> print(gs.best_params_)
# Output: {'svc__C': 10.0, 'svc__gamma': 0.01, 'svc__kernel': 'rbf'}

For more information on evaluation metrics, see Wikipedia - Confusion matrix and [sklearn.metrics](http://scikit-learn.org/stable /modules/classes.html#module-sklearn.metrics).
