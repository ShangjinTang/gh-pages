Principal Component Analysis

Similar to feature selection, we can use feature extraction to reduce the number of features in the dataset. The difference is that feature selection preserves the original features, while feature extraction transforms or projects the data into a new feature space. This article introduces Principal Component Analysis (PCA), which is an unsupervised linear transformation technology whose main features are feature extraction and dimensionality reduction.

Algorithm Principle​

PCA is widely used in different fields and helps us identify patterns in data based on correlations between features. The principle is to find the direction of maximum variance in high-dimensional data and project it onto a new orthogonal space with the same or fewer dimensions than the original.

The orthogonal axes (i.e. principal components) of the new subspace can be interpreted as the direction of the maximum variance under the constraint that the new feature axes are orthogonal to each other, as shown in the following figure:

In the above figure, x1x_1x1​ and x2x_2x2​ are the original feature axes, and PC1 and PC2 are the first and second principal components of the data set.

PCA constructs a dimensional transformation matrix W\boldsymbol WW to map the high-dimensional space vector x\boldsymbol xx to the low-dimensional space vector z\boldsymbol zz:

x=[x1,x2,⋯ ,xd],x∈Rd↓xW,W∈Rd×kz=[z1,z2,⋯ ,zk],z∈Rk\begin{align} \boldsymbol x & = [x_1, x_2, \cdots, x_d], x \in R^d \\ & \downarrow \boldsymbol x \boldsymbol W, \boldsymbol W \in R^{d \times k} \\ \boldsymbol z & = [z_1, z_2, \cdots, z_k], z \in R^k \end{align}xz​=[x1​,x2​,⋯,xd​],x∈Rd↓xW,W∈Rd×k=[z1​,z2​,⋯,zk​],z∈Rk​​

After dimensionality transformation, the first principal component will have the largest variance, the second principal component will have the second largest variance, and so on. These principal components must all be uncorrelated (even if there is correlation between input features), and the mathematical representation is that the principal component vectors are orthogonal to each other.

It is important to note that PCA orientation is very sensitive to data scaling. For example, if one feature is evenly distributed between [-100,100] and another feature is evenly distributed between [-1,1], PCA will consider the former feature to be much more important. Therefore, when applying PCA, it is necessary to use normalization to scale the input features to the same scale.

A brief introduction to the algorithm flow of PCA is as follows:

  1. Standardize the input ddd dimensional data set X\boldsymbol XX.
  2. Construct the covariance matrix.
  3. Calculate the eigenvectors and eigenvalues of the covariance matrix.
  4. Sort the eigenvalues in descending order and sort the corresponding eigenvectors.
  5. Extract the eigenvectors corresponding to the kkk maximum eigenvalues, where kkk (k≤dk \le dk≤d) is the dimension of the new feature subspace.
  6. Construct the projection matrix W\boldsymbol WW using the extracted kkk eigenvectors.
  7. Use the projection matrix W\boldsymbol WW to transform X\boldsymbol XX to obtain a new kkk-dimensional feature subspace.

Data Load and Standardization​

Import Wine dataset, the 1st column is its classification label, and the 2nd to 14th columns are features. Split the data set into 70% training set and 30% test set:

import pandas as pd
from sklearn.model_selection import train_test_split

df_wine = pd.read_csv('https://archive.ics.uci.edu/ml/machine-learning-databases/wine/wine.data', header=None)
X, y = df_wine.iloc[:, 1:].values, df_wine.iloc[:, 0].values
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=0, stratify=y)

Standardize the training and test sets:

from sklearn.preprocessing import StandardScaler

sc = StandardScaler()
sc.fit(X_train)
X_train_std = sc.transform(X_train)
X_test_std = sc.transform(X_test)

Construct Covariance Matrix​

Next, the covariance matrix is introduced. For the two feature vectors of nnn (usually represented by column vectors in machine learning) xj\boldsymbol x_jxj​ and xk\boldsymbol x_kxk​, their covariance is as follows:

σjk=σkj=1n∑i=1n(xj(i)−μj)(xk(i)−μk)\begin{align} \sigma_{jk} = \sigma_{kj} = {\frac 1 n} \sum_{i=1}^n (x_j^{(i)} - \mu_j)(x_k^{(i)} - \mu_k ) \end{align}σjk​=σkj​=n1​i=1∑n​(xj(i)​−μj​)(xk(i)​−μk​)​​

where μj\mu_jμj​ and μk\mu_kμk​ represent the average values of xj\boldsymbol x_jxj​ and xk\boldsymbol x_kxk​ respectively. For the sample of PCA, both means are 0 after normalization.

Covariance σjk\sigma_{jk}σjk​ can represent the degree of correlation between xj\boldsymbol x_jxj​ and xk\boldsymbol x_kxk​:

  • If σjk\sigma_{jk}σjk​ is greater than 0, it indicates that the two features are positively correlated, that is, one feature becomes larger as the other feature becomes larger.
  • If σjk\sigma_{jk}σjk​ is greater than 0, it indicates that the two features are negatively correlated, that is, one feature becomes smaller as the other feature becomes larger.
  • If σjk\sigma_{jk}σjk​ is equal to 0, it indicates that the two features are uncorrelated, that is, changes in one feature are not affected by changes in the other feature. At this time, the two eigenvectors are orthogonal to each other.

Next build the covariance matrix Σ\boldsymbol \SigmaΣ (this is the capital Greek letter Sigma, not to be confused with the summation symbol ∑\sum∑). Taking 3 features as an example, their covariance matrix is as follows:

Σ=[σ12σ12σ13σ21σ22σ23σ31σ32σ32]\begin{align} \boldsymbol \Sigma = \begin{bmatrix} \sigma_1^2 & \sigma_{12} & \sigma_{13} \\ \sigma_{21} & \sigma_2^2 & \sigma_{23} \\ \sigma_{31} & \sigma_{32} & \sigma_3^2 \end{bmatrix} \end{align}Σ=​σ12​σ21​σ31​​σ12​σ22​σ32​​σ13​σ23​σ32​​​​​

The covariance matrix Σ\boldsymbol \SigmaΣ is a real symmetric matrix. The diagonal of the matrix is the variance of the features, and the rest is the covariance between features. The eigenvector of Σ\boldsymbol \SigmaΣ represents the principal component (that is, the direction with the largest variance), and the corresponding eigenvalue is the size of the principal component.

The Wine dataset has 13 features, and we will obtain 13 eigenvectors and eigenvalues from a 13×13 dimensional covariance matrix.

Eigenvalues and Eigenvectors​

Solve the eigenvalue λ\lambdaλ and eigenvector v\boldsymbol vv of the matrix to satisfy the conditions:

Σv=λv\begin{align} {\boldsymbol \Sigma} {\boldsymbol v} = \lambda {\boldsymbol v} \end{align}Σv=λv​​

The mathematical calculation of matrix eigenvalues and eigenvectors is relatively complicated and will not be introduced here.

Eigenvalues and eigenvectors can be easily calculated using numpy’s built-in linalg.eig function:

import numpy as np

cov_mat = np.cov(X_train_std.T)
eigen_vals, eigen_vecs = np.linalg.eig(cov_mat)

The above creates a covariance matrix through numpy.cov, and then performs eigendecomposition on it through numpy.linalg.eig, resulting in a vector eigen_vals composed of 13 eigenvalues and 13 eigenvectors (column vectors) Matrix eigen_vecs.

Explained Variance Ratio​

To compress ddd-dimensional space into kkk-dimensional space, you need to select kkk feature vectors (principal components) that contain the most information (maximum variance). The eigenvalues reflect the size of the eigenvector. It is necessary to sort the eigenvalues in descending order, and then extract the eigenvectors corresponding to the first kkk eigenvalues.

Before extracting features, in order to observe the difference between the variances of the features, we first plot the explained variance ratio of the feature values:

import matplotlib.pyplot as plt

tot = sum(eigen_vals)
var_exp = [(i / tot) for i in sorted(eigen_vals, reverse=True)]
cum_var_exp = np.cumsum(var_exp)

plt.bar(range(1, 14), var_exp, alpha=0.5, align='center', label='individual explained variance')
plt.step(range(1, 14), cum_var_exp, where='mid', label='cumulative explained variance')
plt.ylabel('Explained variance ratio')
plt.xlabel('Principal component index')
plt.legend(loc='best')
plt.show()

It can be seen that the first principal component accounts for approximately 40% of the variance, the second principal component accounts for approximately 20% of the total variance, and these two eigenvalues explain close to 60% of the variance of the data set.

It should be noted that y_train is not used in the entire process. PCA is unsupervised learning and class labels are never used in the process of analyzing and extracting principal components.

Feature Extraction​

This section uses the Wine data set to demonstrate the final steps of feature extraction:

  • Extract the eigenvectors corresponding to the kkk largest eigenvalues, where kkk (k≤dk \le dk≤d) is the dimension of the new feature subspace.
  • Construct the projection matrix W\boldsymbol WW using the extracted kkk eigenvectors,
  • Use the projection matrix W\boldsymbol WW to transform X\boldsymbol XX to obtain a new kkk-dimensional eigensubspace.

Simply put, it is to sort the eigenvectors in descending order of eigenvalues, construct a projection matrix from the eigenvectors, and use the projection matrix to transform the data into a low-dimensional subspace.

Feature vector sorting:

# Make a list of (eigenvalue, eigenvector) tuples
eigen_pairs = [(np.abs(eigen_vals[i]), eigen_vecs[:, i]) for i in range(len(eigen_vals))]
# Sort the (eigenvalue, eigenvector) tuples from high to low
eigen_pairs.sort(key=lambda k: k[0], reverse=True)

Taking two principal components as an example, construct the projection matrix:

Note that for the purpose of drawing a two-dimensional scatter plot later, only two principal components are selected here. In practice, the number of principal components should be determined based on the balance of efficiency and performance.

w = np.hstack((eigen_pairs[0][1][:, np.newaxis], eigen_pairs[1][1][:, np.newaxis]))

Transform the original dataset using a projection matrix:

x′=xW↓X′=XW\begin{align} \boldsymbol x' & = \boldsymbol x \boldsymbol W \\ & \downarrow \\ \boldsymbol X' & = \boldsymbol X \boldsymbol W \end{align}x′X′​=xW↓=XW​​

Where X\boldsymbol XX is the original data set, x\boldsymbol xx is a single sample; X′\boldsymbol X'X′ is the transformed data set, and x′\boldsymbol x'x′ is a single sample.

X_train_pca = X_train_std.dot(w)

The original Wine data set has 13 dimensions, which is reduced to 2 dimensions through the feature matrix W∈R13×2\boldsymbol W \in R^{13 \times 2}W∈R13×2. The reduced dimension data set is plotted:

colors = ['r', 'b', 'g']
markers = ['s', 'x', 'o']
for l, c, m in zip(np.unique(y_train), colors, markers):
     plt.scatter(X_train_pca[y_train == l, 0], X_train_pca[y_train == l, 1], c=c, label=l, marker=m)
plt.xlabel('PC 1')
plt.ylabel('PC 2')
plt.legend(loc='lower left')
plt.show()

As can be seen from the figure below, the data is more dispersed along the xxx axis (the first principal component) than along the yyy axis (the second principal component), which is consistent with the larger variance of the first principal component in the previous section. . In addition, the classified data after dimensionality reduction still retains its characteristics and can be classified using a linear classifier.

It needs to be emphasized again that PCA is an unsupervised technique and does not use any class label information during the entire PCA process.

Scikit-learn Implementation​

Boundary plotting function plot_decision_regions See previous article.

Reduce the test set and training set to two dimensions through sklearn.decomposition.PCA, and then use logistic regression to reduce The post-dimensional test set is linearly classified:

from sklearn.linear_model import LogisticRegression
from sklearn.decomposition import PCA

pca = PCA(n_components=2)
lr = LogisticRegression()
X_train_pca = pca.fit_transform(X_train_std)
X_test_pca = pca.transform(X_test_std)
lr.fit(X_train_pca, y_train)
plot_decision_regions(X_train_pca, y_train, classifier=lr)
plt.xlabel('PC 1')
plt.ylabel('PC 2')
plt.legend(loc='lower left')
plt.show()

Draw the decision boundary obtained through training in the reduced dimension space and observe whether it can be classified well:

plot_decision_regions(X_test_pca, y_test, classifier=lr)
plt.xlabel('PC1')
plt.ylabel('PC2')
plt.legend(loc='lower left')
plt.show()

It can be seen that logistic regression performs well on the new two-dimensional subspace, with very few sample classification errors on the test data set.

Regarding the proportion of explained variance of the data set, you can set the parameter n_components of PCA to None, and obtain it through the explained_variance_ratio_ attribute after transforming the test set:

pca = PCA(n_components=None)
X_train_pca = pca.fit_transform(X_train_std)
print(pca.explained_variance_ratio_)
