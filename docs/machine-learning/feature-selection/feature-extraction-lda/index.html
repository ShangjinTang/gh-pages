Linear Discriminant Analysis

Linear Discriminant Analysis (LDA), as a feature extraction technology, can improve computational efficiency and prevent overfitting due to the curse of dimensionality. LDA is similar to PCA, except that the goal of PCA is to find the orthogonal component axes that maximize the variance in the data set, while the goal of LDA is to find the feature subspace that optimizes class separability.

LDA and PCA are linear transformation techniques that can be used to reduce the number of dimensions in a data set. The difference is that LDA is a supervised algorithm while PCA is an unsupervised algorithm.

LDA aims to find the linear discriminant corresponding to the most projections, as shown in the figure below, where circles and crosses represent two classes of normal distribution samples respectively:

The linear discriminant shown on the xxx axis (LD1) classifies well; while the linear discriminant shown on the yyy axis (LD2) does not capture any classification information, although it has a larger projected variance.

It is assumed in LDA that the data is normally distributed. Furthermore, we assume that the covariance matrices of the classes are the same and the features are uncorrelated with each other.

Algorithm Principle​

A brief introduction to the algorithm flow of LDA is as follows:

  1. Standardize the input ddd dimensional data set X\boldsymbol XX.
  2. Calculate the ddd-dimensional mean vector of each class, and construct the intra-class scatter matrix SW\boldsymbol S_WSW​ and the inter-class scatter matrix SB\boldsymbol S_BSB​.
  3. Calculate the eigenvectors and eigenvalues of the matrix SW−1SB\boldsymbol S_W^{-1} \boldsymbol S_BSW−1​SB​.
  4. Sort the eigenvalues in descending order and sort the corresponding eigenvectors.
  5. Extract the eigenvectors corresponding to the kkk maximum eigenvalues, where kkk (k≤dk \le dk≤d) is the dimension of the new feature subspace.
  6. Construct the projection matrix W\boldsymbol WW using the extracted kkk eigenvectors.
  7. Use the projection matrix W\boldsymbol WW to transform X\boldsymbol XX to obtain a new kkk-dimensional feature subspace.

LDA is very similar to PCA, by decomposing the matrix into eigenvalues and eigenvectors, and then obtaining a new low-dimensional feature space by projecting the matrix. The difference is that LDA takes class labels into account (step 2).

Data Load and Standardization​

Import Wine dataset, the 1st column is its classification label, and the 2nd to 14th columns are features. Split the data set into 70% training set and 30% test set:

import pandas as pd
from sklearn.model_selection import train_test_split

df_wine = pd.read_csv('https://archive.ics.uci.edu/ml/machine-learning-databases/wine/wine.data', header=None)
X, y = df_wine.iloc[:, 1:].values, df_wine.iloc[:, 0].values
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=0, stratify=y)

Standardize the training and test sets:

from sklearn.preprocessing import StandardScaler

sc = StandardScaler()
sc.fit(X_train)
X_train_std = sc.transform(X_train)
X_test_std = sc.transform(X_test)

Construct Scatter Matrix​

Mean Vector​

Calculate the mean vector, which will be used later to construct the within-class scatter matrix and the between-class scatter matrix. In the mean vector mi\boldsymbol m_imi​ of the iiith class sample, the average value μm\mu_mμm​ of the sample features of this class is stored:

mi=1ni∑x∈Dicxm\begin{align} \boldsymbol m_i = \frac 1 {n_i} \sum_{x \in D_i}^c \boldsymbol x_m \end{align}mi​=ni​1​x∈Di​∑c​xm​​​

Get three mean vectors:

mi=[μi,alcoholμi,malic acid⋮μi,proline]\begin{align} \boldsymbol m_i = \begin{bmatrix} \mu_{i,alcohol} \\ \mu_{i,malic \ acid} \\ \vdots \\ \mu_{i,proline} \end{bmatrix} \end{align}mi​=​μi,alcohol​μi,malic acid​⋮μi,proline​​​​​

Intra-Class Scatter Matrix​

The intra-class scatter matrix SW\boldsymbol S_WSW​ is calculated as follows, where Si\boldsymbol S_iSi​ is the scatter matrix of class iii.

SW=∑i=1cSiSi:=1niSi=1ni∑x∈Dic(x−mi)(x−mi)T=Σi\begin{align} \boldsymbol S_W & = \sum_{i=1}^c \boldsymbol S_i \\ \boldsymbol S_i & := \frac 1 {n_i} \boldsymbol S_i = \frac 1 {n_i} \sum_{x \in D_i}^c (\boldsymbol x - \boldsymbol m_i) (\boldsymbol x - \boldsymbol m_i) ^T = \boldsymbol \Sigma_i \end{align}SW​Si​​=i=1∑c​Si​:=ni​1​Si​=ni​1​x∈Di​∑c​(x−mi​)(x−mi​)T=Σi​​​

​ Since LDA assumes that the class labels in the training set are uniformly distributed, here the hash matrix Si\boldsymbol S_iSi​ is scaled to 1ni\frac 1 {n_i}ni​1​, which happens to be the covariance matrix after scaling.

d = 13 # number of features
S_W = np.zeros((d, d))
for label,mv in zip(range(1, 4), mean_vecs):
     class_scatter = np.cov(X_train_std[y_train==label].T)
     S_W += class_scatter
print('Scaled within-class scatter matrix: {}x{}'.format(S_W.shape[0], S_W.shape[1]))
# Output: Scaled within-class scatter matrix: 13x13

Inter-Class Scatter Matrix​

The inter-class scatter matrix SB\boldsymbol S_BSB​ is calculated as follows, where m\boldsymbol mm is the mean vector of all samples:

SB=∑i=1cni(mi−m)(mi−m)T\begin{align} \boldsymbol S_B = \sum_{i=1}^cn_i (\boldsymbol m_i - \boldsymbol m) (\boldsymbol m_i - \boldsymbol m)^T \end{align}SB​=i=1∑c​ni​(mi​−m)(mi​−m)T​​ 
mean_overall = np.mean(X_train_std, axis=0)
d = 13 # number of features
S_B = np.zeros((d, d))
for i, mean_vec in enumerate(mean_vecs):
     n = X_train[y_train == i + 1, :].shape[0]
     mean_vec = mean_vec.reshape(d, 1) # make column vector
     mean_overall = mean_overall.reshape(d, 1)
     S_B += n * (mean_vec - mean_overall).dot((mean_vec - mean_overall).T)
print('Between-class scatter matrix: {}x{}'.format(S_B.shape[0], S_B.shape[1]))
# Output: Between-class scatter matrix: 13x13

Eigenvalues and Eigenvectors​

Calculate the eigenvalues and eigenvectors of the matrix SW−1SB\boldsymbol S_W^{-1} \boldsymbol S_BSW−1​SB​ and sort the corresponding eigenvectors in descending order of eigenvalues:

eigen_vals, eigen_vecs = np.linalg.eig(np.linalg.inv(S_W).dot(S_B))
eigen_pairs = [(np.abs(eigen_vals[i]), eigen_vecs[:, i]) for i in range(len(eigen_vals))]
eigen_pairs.sort(key=lambda k: k[0], reverse=True)

In LDA, the number of linear discriminants is at most c−1c-1c−1, where ccc is the number of class labels.

Discrimination Ability of Linear Discriminant​

To measure how much class-discriminative information a linear discriminant (eigenvector) captures, plot the discriminability of different linear discriminants:

import matplotlib.pyplot as plt

tot = sum(eigen_vals.real)
discr = [(i / tot) for i in sorted(eigen_vals.real, reverse=True)]
cum_discr = np.cumsum(discr)

plt.bar(range(1, 14), discr, alpha=0.5, align='center', label='individual "discriminability"')
plt.step(range(1, 14), cum_discr, where='mid', label='cumulative "discriminability"')
plt.ylabel('"discriminability" ratio')
plt.xlabel('Linear Discriminants')
plt.ylim([-0.1, 1.1])
plt.legend(loc='best')
plt.show()

Transform the original dataset using a projection matrix:

x′=xW↓X′=XW\begin{align} \boldsymbol x' & = \boldsymbol x \boldsymbol W \\ & \downarrow \\ \boldsymbol X' & = \boldsymbol X \boldsymbol W \end{align}x′X′​=xW↓=XW​​ 
w = np.hstack((eigen_pairs[0][1][:, np.newaxis].real, eigen_pairs[1][1][:, np.newaxis].real))
X_train_lda = X_train_std.dot(w)

Scikit-learn Implementation​

Boundary plotting function plot_decision_regions See previous article.

Reduce the test set and training set to two dimensions through sklearn.discriminant_analysis.LinearDiscriminantAnalysis, and then use logistic regression to reduce The post-dimensional test set is linearly classified:

from sklearn.linear_model import LogisticRegression
from sklearn.discriminant_analysis import LinearDiscriminantAnalysis as LDA

lda = LDA(n_components=2)
lr = LogisticRegression()
X_train_lda = lda.fit_transform(X_train_std, y_train)
X_test_lda = lda.transform(X_test_std)
lr.fit(X_train_lda, y_train)
plot_decision_regions(X_train_lda, y_train, classifier=lr)
plt.xlabel('LD 1')
plt.ylabel('LD 2')
plt.legend(loc='lower left')
plt.show()

Draw the decision boundary obtained through training in the reduced dimension space and observe whether it can be classified well:

plot_decision_regions(X_test_lda, y_test, classifier=lr)
plt.xlabel('LD 1')
plt.ylabel('LD 2')
plt.legend(loc='lower left')
plt.show()

It can be seen that logistic regression performs well on the new two-dimensional subspace, with very few sample classification errors on the test data set.
